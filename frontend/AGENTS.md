# Frontend Review Guide

## Review guidelines

- 리뷰 코멘트는 한국어로 작성한다.
- 실제 버그, 인증 상태 꼬임, 데이터 유실 가능성, 접근성 문제, 테스트 누락을 우선 지적한다.
- 단순 스타일 취향이나 nit는 실제 결함 가능성이 있을 때만 지적한다.
- 로그인 상태 판단이 단순 토큰 존재 여부에만 의존해 만료 토큰을 로그인으로 오인하지 않는지 확인한다.
- Authorization 헤더와 `X-Session-Key` 헤더가 로그인/비로그인/토큰 만료 흐름에서 안전하게 전달되는지 확인한다.
- React state가 로그인, 로그아웃, 새 질문, 스트리밍 중단, 삭제 모달, 이전 대화 열기 흐름에서 꼬이지 않는지 확인한다.
- 파괴적 동작인 삭제, 로그아웃, 문서 삭제는 확인 단계나 명확한 피드백이 있는지 확인한다.
- 모달, 팝오버, 드로어는 Escape, 외부 클릭, focus 이동, `aria-*` 속성이 적절한지 확인한다.
- API 오류가 사용자에게 이해 가능한 메시지로 표시되는지 확인한다.
- EventSource/SSE 연결은 중단, 오류, 완료, 늦게 도착한 이벤트 처리에서 중복 state update가 없는지 확인한다.
- 화면 변경은 비로그인 사용자와 로그인 사용자 양쪽 레이아웃을 모두 확인한다.

## Clean code checklist

- 컴포넌트가 너무 많은 책임을 가지면 작은 컴포넌트나 hook으로 분리할 수 있는지 검토한다.
- API 호출 코드는 `services/` 계층에 두고 화면 컴포넌트에 fetch 세부 구현을 직접 넣지 않는다.
- 반복되는 표시 포맷, role 라벨, 식별자 라벨은 helper로 분리한다.
- boolean state가 많아지면 서로 모순되는 조합이 생기지 않는지 확인한다.
- 전역 `button` 스타일이 icon button, modal button, admin button에 의도치 않게 섞이지 않는지 확인한다.
- hover 색상은 과도한 브랜드색 배경보다 옅은 회색 피드백을 우선한다.

## Project-specific focus

- 채팅 화면은 인하공업전문대학 홈페이지 방문자용 AI 안내 맥락을 유지한다.
- 비로그인 사용자는 채팅 이력 사이드바를 보지 않는다.
- 로그인 사용자는 자신의 채팅 이력만 보고 삭제할 수 있어야 한다.
- 관리자 프로필 표시에서는 학생/교직원/관리자 식별자 라벨 확장 가능성을 유지한다.
- 문서 업로드 UI는 drag/drop 기본 브라우저 동작, 같은 파일 재선택, 업로드 중 삭제 같은 edge case를 확인한다.
